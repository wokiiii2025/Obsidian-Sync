# Obsidian 插件使用说明

本文说明 Zero Knowledge Sync 插件的常用设置、附件管理方式，以及后续可扩展功能建议。

## 一、基础使用流程

1. 将插件文件放入 Obsidian Vault 的插件目录：

```text
<Vault>/.obsidian/plugins/obsidian-zero-knowledge-sync/
```

需要包含：

- `manifest.json`
- `main.js`
- `styles.css`

2. 在 Obsidian 中启用插件。
3. 打开插件设置。
4. 填写同步服务器地址。
5. 注册新 Vault，或登录已有 Vault。
6. 输入 Vault 密码解锁。
7. 点击右下角状态栏同步图标，或开启定时同步。

## 二、同步行为

插件支持以下同步方式：

- 手动同步：点击 Obsidian 右下角同步图标。
- 定时同步：开启后按设置的间隔自动同步。
- 文件变化后同步：本地文件创建、修改、删除或重命名后，插件会延迟触发同步。

当前默认文件变化延迟为 60 秒，避免频繁编辑时连续触发同步。

设置页会显示同步状态和统计信息，包括：

- 当前状态：空闲、同步中、成功、失败、已锁定。
- 已跟踪文件数量。
- 上传、下载、冲突数量。
- 最近同步开始和完成时间。
- 最近错误信息。
- 最近 20 条同步历史，设置页默认展示最近 5 条。

## 三、附件同步

插件不仅同步 `.md` 笔记，也同步图片、PDF、音频、视频、压缩包等附件。

附件会和笔记一样在本地加密后上传，服务端不会看到明文文件名和明文内容。

## 四、附件自动整理

开启“管理附件”后，插件会把非 Markdown 文件移动到统一附件目录，并自动改写笔记里的引用路径。

初次启用插件时，如果 Vault 中已经存在未统一管理的附件，插件会弹出确认提示。确认后才会批量整理已有附件；取消后不会自动移动已有文件。之后也可以在设置页点击“整理已有附件”手动执行。

支持的引用格式：

```markdown
![[photo.png]]
[[document.pdf]]
![photo](photo.png)
[document](document.pdf)
```

### 4.1 设置项说明

| 设置项 | 说明 |
|---|---|
| 管理附件 | 是否启用附件自动移动和链接改写 |
| 附件目录 | 附件基础目录，例如 `Attachments` |
| 附件整理方式 | 决定附件目录下如何分组 |
| 附件日期格式 | 按日期分组时使用的日期路径格式 |
| 附件类型目录映射 | 自定义扩展名应该进入哪个类型目录 |
| 整理已有附件 | 手动把历史附件迁移到当前附件目录规则 |

### 4.2 附件整理方式

| 整理方式 | 示例结果 |
|---|---|
| 单一目录 | `Attachments/photo.png` |
| 按类型 | `Attachments/images/photo.png` |
| 按日期 | `Attachments/2026-06-24/photo.png` |
| 按类型和日期 | `Attachments/images/2026-06-24/photo.png` |

### 4.3 日期格式示例

| 日期格式 | 生成路径片段 |
|---|---|
| `YYYY/MM/DD` | `2026/06/24` |
| `YYYY-MM-DD` | `2026-06-24` |
| `YYYY_MM_DD` | `2026_06_24` |
| `YYYY/MM` | `2026/06` |

如果你希望图片路径是：

```text
Attachments/images/2026-06-24/photo.png
```

请这样设置：

```text
附件目录：Attachments
附件整理方式：按类型和日期
附件日期格式：YYYY-MM-DD
```

如果你希望路径是：

```text
Attachments/images/2026/06/24/photo.png
```

请这样设置：

```text
附件目录：Attachments
附件整理方式：按类型和日期
附件日期格式：YYYY/MM/DD
```

### 4.4 类型目录映射

默认类型目录如下，也可以在设置页的“附件类型目录映射”中修改：

| 类型目录 | 文件扩展名 |
|---|---|
| `images` | `png`, `jpg`, `jpeg`, `gif`, `webp`, `svg`, `bmp`, `avif` |
| `documents` | `pdf`, `doc`, `docx`, `ppt`, `pptx`, `xls`, `xlsx`, `csv` |
| `audio` | `mp3`, `wav`, `m4a`, `flac`, `ogg`, `aac` |
| `video` | `mp4`, `mov`, `mkv`, `webm`, `avi` |
| `archives` | `zip`, `rar`, `7z`, `tar`, `gz` |
| `files` | 其他文件类型 |

映射格式为每行一个目录：

```text
images: png, jpg, jpeg, gif, webp
documents: pdf, docx, xlsx
screenshots: png
```

如果一个扩展名同时出现在多行，优先使用靠前的映射。

### 4.5 孤立附件扫描

设置页提供“孤立附件”扫描按钮。

扫描逻辑：

- 只扫描附件目录下的非 Markdown 文件。
- 读取 Vault 中的 Markdown 笔记。
- 识别 `![[...]]`、`[[...]]`、`![...](...)`、`[...](...)` 这几类引用。
- 如果附件路径和文件名都没有被任何笔记引用，就标记为孤立附件。

扫描结果会保存到插件设置中，并在设置页展示前 5 个结果。

如果确认这些文件不再需要，可以点击“清理”。清理前会再次弹窗确认，确认后会把孤立附件移动到系统回收站，而不是直接永久删除。

## 五、语言切换

插件设置支持中文和英文切换。

切换语言后，设置页中的文本、按钮、状态提示会使用对应语言。

## 六、可以继续扩展的实用功能

| 优先级 | 功能 | 价值 |
|---|---|---|
| 高 | 冲突可视化处理 | 在 Obsidian 中对比两个版本并选择保留、合并或另存 |
| 中 | 文件夹级选择同步 | 某些目录同步，某些目录不同步，适合大 Vault 和隐私目录 |
| 中 | 大附件分片上传 | 大 PDF、视频、音频同步更稳定，失败后可续传 |
| 中 | 设备管理 | 显示已登录设备，支持撤销某台设备 |
| 中 | 同步前预览 | 展示即将上传/下载/删除的文件列表，适合谨慎用户 |
| 中 | 附件重命名策略 | 同名文件可按哈希、日期、序号等方式命名 |
| 低 | 加密备份导出/导入 | 方便迁移服务器或灾难恢复 |
| 低 | Web 管理后台 | 查看服务健康状态、用户、Vault、设备、存储占用 |
| 低 | 移动端专项优化 | 针对 iOS/Android 的后台同步限制、文件 API 差异做适配 |

已支持：

- 同步历史面板。
- 自定义类型目录映射。
- 孤立附件检查。

建议下一步优先做：

1. 冲突可视化处理。
2. 文件夹级选择同步。
3. 大附件分片上传。

这几项最贴近日常使用，也最能减少后续排错成本。
